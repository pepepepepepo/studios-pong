### 開発時の注意事項（トラブルシューティング）

#### サーバー起動時のターミナル管理
**問題**: uvicornをバックグラウンド起動後、同じターミナルで別のコマンド（curl、Invoke-WebRequestなど）を実行するとサーバーが停止する

**原因**: PowerShellのジョブ管理とVS Codeターミナル統合の仕様により、同一セッション内でのバックグラウンドプロセスと新規コマンドが干渉

**解決策**:
1. **サーバー専用ターミナルを作る**: uvicorn起動後、そのターミナルは触らない
2. **別のターミナルを開く**: テストやcurlは新しいターミナルで実行
3. **運用ルール**: 「Terminal A = サーバー専用、Terminal B = 作業用」と決める

**良い例**:
```powershell
# Terminal A (サーバー専用)
cd F:\saijinos
.venv\Scripts\python.exe -m uvicorn main:app --port 8000 --reload
# ← このターミナルはこのまま放置

# Terminal B (新しいターミナルで作業)
curl http://localhost:8000/health  # ← 問題なし✅
```
